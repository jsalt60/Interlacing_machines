This repository contains the supplementary material used in the study on reducing the computational cost of autonomous vehicle front steering control by means of interlacing techniques, presented in Vicente Roig, J., & Salt, J. (2026).�Autonomous Vehicle Front Steering Control Computation Saving

## Repository file list

Below we describe the main files in the repository, grouped by file type.

### `.mlx` files (MATLAB Live Scripts)

- ControlYawN2Zhang.mlx: Live Script that implements the interlacing computations and the yaw/lateral control simulation of the vehicle for the case with N = 2 interlacing intervals.
- ControlYawN3Zhang.mlx: Version for N = 3, which computes the interlacing parameters and runs the corresponding yaw control simulations.
- ControlYawN4Zhang.mlx: Script for the case N = 4, used to evaluate the behaviour of the interlaced controller versus the original controller.
- ControlYawN5Zang.mlx: Live Script for N = 5, which generates the reference trajectories, applies the interlacing scheme and records the system response signals.
- ControlYawN6Zang.mlx: Version for N = 6, aimed at studying the trade-off between computational saving and performance degradation.
- ControlYawN8Zang.mlx: Case with N = 8, illustrating a more aggressive interlacing degree and its effects on vehicle guidance.
- ControlYawZhangCompN.mlx: Comparative script that gathers and compares the results for different values of N, facilitating the joint analysis of simulations and computational savings.


### `.mat` files (configurations and results)

- InterlacinN2.mat: Data associated with the interlacing configuration and results for N = 2.
- InterlacinN3.mat: Data corresponding to the case N = 3.
- InterlacinN4.mat: Data for the interlacing configuration with N = 4.
- InterlacinN5.mat: Data generated in the simulations with N = 5.
- InterlacinN6.mat: Data for the case N = 6.
- InterlacinN8.mat: Data associated with the interlacing configuration with N = 8.
- regmu_cochezhang.mat: File containing the controller/regulator gains for the vehicle (Zhang model), used as the basis for the different interlacing configurations.

### `.slx` and `.slxc` files (Simulink models and compiled builds)

Simulink models:

- PruebaN2.slx: Simulink model used for the simulations of the case N = 2, implementing the interlaced controller and the vehicle model.
- PruebaN3.slx: Simulink model for the tests with N = 3.
- PruebaN4.slx: Simulink model for the tests with N = 4.
- PruebaN5.slx: Simulink model for the tests with N = 5.
- PruebaN6.slx: Simulink model for the tests with N = 6.
- PruebaN7.slx: Simulink model for the tests with N = 7.
- PruebaN8.slx: Simulink model for the tests with N = 8.

Compiled Simulink artifacts:

- PruebaN2.slxc: Compiled file generated by Simulink to accelerate the simulations of the PruebaN2.slx model.
- PruebaN3.slxc: Compiled file associated with PruebaN3.slx.
- PruebaN5.slxc: Compiled file associated with PruebaN5.slx.
- PruebaN6.slxc: Compiled file associated with PruebaN6.slx.
- PruebaN8.slxc: Compiled file associated with PruebaN8.slx.

### `.m` files (MATLAB functions)

- regdiagsep.m: MATLAB function that separates a discrete-time system into several subspaces after transforming it to a (almost) modal diagonal form and ordering the poles from fastest to slowest. It generates state-space subsystems useful for multirate or decentralized control designs.
- regsep.m: MATLAB function that separates a system into subspaces directly from the original state-space realization, adding the coupling terms between state blocks as additional inputs and returning each subspace as a state-space system.
